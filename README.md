## 視覚要素作成ガイドライン

### SVG図表作成の重要原則
**【グラフ位置問題の修正】**
SVG座標系とメモリ内理解の乖離を防ぐため:

1. **座標の明示的計算**
```xml
<!-- 悪い例: 直感的な配置 -->
<rect x="100" y="50" ... />  <!-- 実際の位置を誤認識 -->

<!-- 良い例: 計算を明示 -->
<!-- グラフ原点(50,200), スケール:1unit=20px -->
<rect x="50" y="180" ... />  <!-- y=200-20*1 で1単位上 -->
```

2. **座標系の文書化**
各SVG図表の前にコメントで座標系を明記:
```html
<!-- 
座標系定義:
- 原点: (50, 250)
- X軸: 右方向, 1単位=30px
- Y軸: 上方向, 1単位=40px (SVGは下が正なので減算)
- 例: データ値(2,3) → SVG座標(110, 130)
-->
<svg>...</svg>
```

3. **数値の視覚的検証**
グラフ作成後、主要データポイントのSVG座標を逆算して確認:
```python
# 検証スクリプト例
def verify_svg_coordinates():
    origin_x, origin_y = 50, 250
    scale_x, scale_y = 30, 40
    data_point = (2, 3)
    svg_x = origin_x + data_point[0] * scale_x  # 110
    svg_y = origin_y - data_point[1] * scale_y  # 130
    print(f"データ{data_point} → SVG({svg_x},{svg_y})")
```

4. **軸ラベルとの整合性確認**
軸ラベルの数値とグラフ要素の位置が一致することを確認
