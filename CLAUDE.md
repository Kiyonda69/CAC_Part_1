# CLAUDE.md - プロジェクトガイドライン

## プロジェクト概要

航空大学校入学試験「総合Part I」科目の思考パズル問題を作成するプロジェクト。
論理的推論、空間視覚化、パターン認識、数学的思考を測定する問題を作成する。
Python検証により解の一意性を保証する。

## 言語

- 主要言語: **日本語**
- 問題内容、解説、ドキュメントはすべて日本語で記述

## 測定能力

- 論理推論
- 空間視覚化
- パターン認識
- 数学的思考
- 制約充足
- 資料読み取り

## 問題仕様

### 作成ルール
- **2問1セット**: 標準難度（問1）と高難度（問2）をペアで作成
- **出力形式**: 2問1セットで1つのHTMLファイルを生成
- **ファイル命名**: `航大思考{N}.html`（Nはセット番号: 1, 2, 3, ...）
  - 例: 航大思考1.html（問1・問2）、航大思考2.html（問1・問2）...

### 難易度別仕様

| 項目 | 問題1（標準） | 問題2（高難度） |
|------|-------------|----------------|
| 配点 | 6点 | 6点 |
| 制限時間 | 3分 | 5分 |
| 難易度 | ★★★★★ | ★★★★★★★★ |
| 解答時間目安 | 170秒 | 310秒 |

- 問題2は問題1の派生・高難度版として作成

## 作成ワークフロー

1. **設計**: 問題タイプを選択（制約充足、暗号解読、空間推論、パターン認識、データ分析）
2. **Python検証**: 総当たりで解の一意性を確認
3. **正解番号ランダム化**: `python3 -c "import random; print(random.randint(1, 5))"` を実行
4. **HTML作成**: `template.html` をベースに、指定セクションのみ編集
5. **問題一覧更新**: `問題一覧.md` に新規問題の情報を追加（必須）

## 問題一覧の更新ルール

新しい問題セットを作成した際は、**必ず** `問題一覧.md` を更新すること。

### 更新手順

1. **作成済み問題セット表への追加**
   - セット番号、ファイル名、問1・問2の概要を追記
   - 形式: `| N | 航大思考N.html | 問1の概要 | 問2の概要 |`

2. **問題タイプ別分類への追加**
   - 該当するカテゴリに新規問題を追記
   - 新しいカテゴリが必要な場合は追加

3. **統計の更新**
   - 総問題セット数、総問題数を更新
   - 最終更新日を更新

### 更新例

```markdown
## 作成済み問題セット
| 21 | 航大思考21.html | 新規問1の概要 | 新規問2の概要 |

## 統計
- **総問題セット数**: 21セット
- **総問題数**: 42問
- **最終更新日**: YYYY-MM-DD
```

## アーカイブとリセット

### サイクル運用

航大思考20まで作成が完了すると、ユーザーが航大思考1〜20をアーカイブします。
その後、再び航大思考1から新規作成を開始します。

### リセット時の作業

アーカイブ後に航大思考1から再作成を開始する際は、**問題一覧.md をリセット**すること。

1. **問題一覧.md の内容をクリア**
   - 作成済み問題セット表を空にする
   - 問題タイプ別分類を空にする
   - 統計をリセット（セット数: 0、問題数: 0）

2. **新規作成開始**
   - 航大思考1.html から作成を再開
   - 通常の作成ワークフローに従う
   - 問題一覧.md を都度更新

## ファイル構成

| 航大思考{N}.html | 問題出力先（2問1セットで生成） |

| scripts/ | 自動化・検証スクリプト |

## 自動化スクリプト

`scripts/` ディレクトリに自動化・検証用スクリプトを配置しています。

### スクリプト一覧

| スクリプト | 説明 | 使用タイミング |
|-----------|------|---------------|
| `update_index.py` | 問題一覧.mdを自動更新 | 問題作成後 |
| `quality_check.py` | 品質チェック（viewBox、正解番号分布等） | 問題作成後 |
| `verify_template.py` | 解の一意性検証テンプレート | 問題設計時にコピーして使用 |
| `verify_XX.py` | 各問題の検証コード | 問題作成時に作成・実行 |

### 使用方法

#### 1. 問題一覧の自動更新

```bash
python3 scripts/update_index.py
```

- 全ての `航大思考*.html` をスキャンして情報を抽出
- `問題一覧.md` を自動生成（既存の分類情報は保持）
- 正解番号も自動抽出して表示

#### 2. 品質チェック

```bash
# 全ファイルをチェック
python3 scripts/quality_check.py

# 特定ファイルのみチェック
python3 scripts/quality_check.py 航大思考1.html
```

**チェック項目:**
- viewBox属性の有無
- SVGサイズが推奨範囲内か
- 正解番号の分布（偏りがないか）
- 解説セクションの有無
- 問題構造の整合性

#### 3. 解の一意性検証

```bash
# テンプレートをコピーして使用
cp scripts/verify_template.py scripts/verify_21.py

# 編集後に実行
python3 scripts/verify_21.py
```

**テンプレートの種類:**
- 順序配置問題
- グリッド論理演算問題（XOR, AND, OR）
- パターン認識問題
- 制約充足問題（CSP）
- 天秤問題
- 暗号解読問題

### 推奨ワークフロー

1. 問題設計時: `verify_template.py` をコピーして検証コードを作成
2. 問題作成後: `quality_check.py` で品質確認
3. 最終確認後: `update_index.py` で問題一覧を更新

## Git運用

### ブランチ命名規則

Claude Code使用時は、システムが自動的に `claude/{説明}-{セッションID}` 形式のブランチを割り当てます。
セッションIDは自動生成され、変更できません。

**タスク説明部分の命名規則:**
- 問題セット作成時は、タスク説明に `CAC-{N}` を含める（Nはセット番号）
- GitHub Issue作成やタスク指示の際に、この命名を明示する

| 作業内容 | タスク説明に含める文字列 | 生成されるブランチ例 |
|---------|----------------------|-------------------|
| 航大思考3の作成 | `CAC-3` | `claude/CAC-3-xxxxx` |
| 航大思考15の作成 | `CAC-15` | `claude/CAC-15-xxxxx` |
| バグ修正・改善 | 作業内容を簡潔に | `claude/fix-svg-issue-xxxxx` |

**注意:** アンダースコア(`_`)はハイフン(`-`)に置き換えて使用する（URL互換性のため）

## HTML編集ルール

- テンプレート: 必ず `template.html` を使用
- 編集範囲: `<!-- ここから問題セクション -->` から `<!-- 解答・解説セクション -->` の間のみ
- カラー: 白黒のみ（グレースケール可）
- 出力形式: 2問1セットで `航大思考{N}.html` として生成（Nはセット番号）
  - 各ファイルに問1（標準難度）と問2（高難度）を含める

### レイアウト構成

各問題は「問題文 → 図表 → 選択肢 → 解説」の順で配置する。解説は各問題の直下に配置し、まとめて最後に配置しない。

```
問1
├── 問題文
├── 図表
├── 選択肢
└── 解説（問1の解説をここに配置）

問2
├── 問題文
├── 図表
├── 選択肢
└── 解説（問2の解説をここに配置）
```

解説セクションのスタイル:
```html
<div style="margin-top: 30px; padding-top: 20px; border-top: 1px solid #ccc;">
    <!-- 解説内容 -->
</div>
```

## 図表サイズガイドライン（重要）

### コンテンツ幅の制約

テンプレートの実効コンテンツ幅は**約660px**（800px - padding 40px×2 - 余白）。
**すべての図表はこの幅に収まるように設計すること。**

### SVGサイズ規則

| 項目 | 推奨値 | 最大値 |
|------|--------|--------|
| SVG幅 | 500px | 650px |
| SVG高さ | 300px | 500px |

### 必須設定

1. **viewBox属性を必ず設定する**
   - viewBoxを設定することで、CSSのmax-width: 100%が正しく機能する
   - width/height属性とviewBoxの値を一致させる

```html
<!-- 正しい例 -->
<svg width="500" height="300" viewBox="0 0 500 300">

<!-- 悪い例（viewBoxなし - レスポンシブに対応できない） -->
<svg width="500" height="300">
```

2. **余白を確保する**
   - SVG内の描画要素は、端から最低20pxの余白を確保
   - テキストラベルが切れないよう注意

3. **大きな図が必要な場合**
   - 図を分割するか、スケールダウンを検討
   - 必要に応じてoverflow: visibleを使用しない（はみ出しの原因）

### 作成後の確認手順

1. HTMLファイルをブラウザで開く
2. ウィンドウ幅を800px程度に縮小して確認
3. 図が枠内に収まっているか、テキストが切れていないか確認
4. 印刷プレビューでも確認（実際の試験は紙媒体）

## SVG座標ガイドライン

SVG図表作成時は、各SVGの前にコメントで座標系とサイズを文書化する:

```html
<!--
図表仕様:
- SVGサイズ: 500x300px
- viewBox: 0 0 500 300
- 余白: 上下左右20px

座標系定義:
- 原点: (50, 250)
- X軸: 右方向, 1単位=30px
- Y軸: 上方向, 1単位=40px (SVGは下が正なので減算)
-->
```

作成後はPythonで座標を検証し、軸ラベルとグラフ要素の位置が一致することを確認する。

def verify_solution():
    """全ての可能性を検証し、唯一解を確認"""
    valid_solutions = []
    # 全パターンを総当たり
    # 制約条件を適用
    # 解の一意性を確認
    assert len(valid_solutions) == 1, f"解が{len(valid_solutions)}個存在"

## 品質チェックリスト

- [ ] Python検証で解が唯一であることを確認
- [ ] すべての条件が解導出に必要
- [ ] 矛盾する条件が存在しない
- [ ] SVG座標が意図した位置と一致
- [ ] **SVGにviewBox属性が設定されている**
- [ ] **図表が幅660px以内に収まっている（ブラウザで確認）**
- [ ] 表のデータが計算結果と一致
- [ ] 正解番号がランダム化されている
- [ ] 170-310秒で解答可能
- [ ] 複数ステップの推論が必要
- [ ] **問題一覧.md が更新されている**

## 禁止事項

- 視覚的矛盾（グラフ値とテキスト値の不一致）
- 自明な解
- 航空専門知識を必要とする問題設計
- Python検証の省略
- 正解番号の手動選択（特に1番や5番への固定）
- フォーマルでない装飾文字の使用（✓、✗、◯、×などの記号）
