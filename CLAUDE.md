# CLAUDE.md - プロジェクトガイドライン

## プロジェクト概要

航空大学校入学試験「総合Part I」科目の思考パズル問題を作成するプロジェクト。
論理的推論、空間視覚化、パターン認識、数学的思考を測定する問題を作成する。
Python検証により解の一意性を保証する。

## 言語

- 主要言語: **日本語**
- 問題内容、解説、ドキュメントはすべて日本語で記述

## 測定能力

- 論理推論
- 空間視覚化
- パターン認識
- 数学的思考
- 制約充足
- 資料読み取り

## 問題仕様

### 作成ルール
- **2問1セット**: 標準難度（問1）と高難度（問2）をペアで作成
- **2ファイル出力**: 同じ問題内容で印刷用と航大思考用の2つのHTMLファイルを生成
- **作成順序**: 印刷用 → 航大思考用 の順で作成
- **番号連動**: 印刷Nと航大思考Nは同じ問題内容（例: 印刷3と航大思考3は同一問題）

### 出力ファイル

| 種類 | テンプレート | 出力先 | ファイル名 |
|------|-------------|--------|-----------|
| 印刷用 | `template_static.html` | `印刷用/` | `印刷{N}.html` |
| 航大思考 | `template_dynamic.html` | `航大思考問題/` | `航大思考{N}.html` |

例: セット3を作成する場合
- `印刷用/印刷3.html`（問1・問2）
- `航大思考問題/航大思考3.html`（問1・問2）

### 難易度別仕様

| 項目 | 問題1（標準） | 問題2（高難度） |
|------|-------------|----------------|
| 配点 | 6点 | 6点 |
| 制限時間 | 3分 | 5分 |
| 難易度 | ★★★★★ | ★★★★★★★★ |
| 解答時間目安 | 170秒 | 310秒 |

- 問題2は問題1の派生・高難度版として作成

## 作成ワークフロー

1. **設計**: 問題タイプを選択（制約充足、暗号解読、空間推論、パターン認識、データ分析）
2. **Python検証**: 総当たりで解の一意性を確認
3. **正解番号ランダム化**: `python3 -c "import random; print(random.randint(1, 5))"` を実行
4. **印刷用HTML作成**: `template_static.html` をベースに `印刷用/印刷{N}.html` を作成
5. **航大思考HTML作成**: `template_dynamic.html` をベースに `航大思考問題/航大思考{N}.html` を作成（同じ問題内容）
6. **問題一覧更新**: `問題一覧.md` に新規問題の情報を追加（必須）
7. **README更新**: `README.md` の統計情報と更新履歴を更新（必須）

**重要**: 印刷用と航大思考用は同じ問題内容を使用するが、テンプレート構造が異なるため、それぞれのテンプレートに合わせて記述すること。

## アーカイブとリセット

### アーカイブの実施

アーカイブはユーザーの指示があった場合にのみ実施する。セット数の上限はない。
ユーザーの指示に従い、「航大思考問題」フォルダと「印刷用」フォルダをアーカイブする。
アーカイブ後、再び航大思考1・印刷1から新規作成を開始する。

### リセット時の作業

アーカイブ後に再作成を開始する際は、以下の手順を実施すること。

1. **フォルダの準備**
   - 新規に `航大思考問題/` フォルダを作成（アーカイブ済みの場合）
   - 新規に `印刷用/` フォルダを作成（アーカイブ済みの場合）
   - または既存フォルダを空にする

2. **問題一覧.md のリセット**
   - 作成済み問題セット表を空にする
   - 問題タイプ別分類を空にする
   - 統計をリセット（セット数: 0、問題数: 0）

3. **新規作成開始**
   - `印刷用/印刷1.html` と `航大思考問題/航大思考1.html` から作成を再開
   - 通常の作成ワークフローに従う
   - 問題一覧.md を都度更新

### リセット後の問題一覧.md テンプレート


## 自動化スクリプト

`scripts/` ディレクトリに自動化・検証用スクリプトを配置しています。

### スクリプト一覧

| スクリプト | 説明 | 使用タイミング |
|-----------|------|---------------|
| `update_index.py` | 問題一覧.mdを自動更新 | 問題作成後 |
| `quality_check.py` | 品質チェック（viewBox、正解番号分布等） | 問題作成後 |
| `verify_template.py` | 解の一意性検証テンプレート | 問題設計時にコピーして使用 |
| `verify_XX.py` | 各問題の検証コード | 問題作成時に作成・実行 |

### 使用方法

#### 1. 問題一覧の自動更新

```bash
python3 scripts/update_index.py
```

- `航大思考問題/` フォルダ内の全HTMLファイルをスキャンして情報を抽出
- `問題一覧.md` を自動生成（既存の分類情報は保持）
- 正解番号も自動抽出して表示

#### 2. 品質チェック

```bash
# 全ファイルをチェック
python3 scripts/quality_check.py

# 特定ファイルのみチェック
python3 scripts/quality_check.py 航大思考問題/航大思考1.html
```

**チェック項目:**
- viewBox属性の有無
- SVGサイズが推奨範囲内か
- 正解番号の分布（偏りがないか）
- 解説セクションの有無
- 問題構造の整合性

#### 3. 解の一意性検証

```bash
# テンプレートをコピーして使用
cp scripts/verify_template.py scripts/verify_21.py

# 編集後に実行
python3 scripts/verify_21.py
```

**テンプレートの種類:**
- 順序配置問題
- グリッド論理演算問題（XOR, AND, OR）
- パターン認識問題
- 制約充足問題（CSP）
- 天秤問題
- 暗号解読問題

### 推奨ワークフロー

1. 問題設計時: `verify_template.py` をコピーして検証コードを作成
2. 問題作成後: `quality_check.py` で品質確認
3. 最終確認後: `update_index.py` で問題一覧を更新

## Git運用

### ブランチ命名規則

Claude Code使用時は、システムが自動的に `claude/{説明}-{セッションID}` 形式のブランチを割り当てます。
セッションIDは自動生成され、変更できません。

**タスク説明部分の命名規則:**
- 問題セット作成時は、タスク説明に `CAC-{N}` を含める（Nはセット番号）
- GitHub Issue作成やタスク指示の際に、この命名を明示する

| 作業内容 | タスク説明に含める文字列 | 生成されるブランチ例 |
|---------|----------------------|-------------------|
| 航大思考3の作成 | `CAC-3` | `claude/CAC-3-xxxxx` |
| 航大思考15の作成 | `CAC-15` | `claude/CAC-15-xxxxx` |
| バグ修正・改善 | 作業内容を簡潔に | `claude/fix-svg-issue-xxxxx` |

**注意:** アンダースコア(`_`)はハイフン(`-`)に置き換えて使用する（URL互換性のため）

## HTML編集ルール

- カラー: 白黒のみ（グレースケール可）
- 各ファイルに問1（標準難度）と問2（高難度）を含める
- 印刷用と航大思考用で同じ問題内容を使用

### テンプレート別の特徴と編集ルール

#### template_static.html（印刷用）

**特徴:**
- 静的な印刷用テンプレート
- A4サイズのページ形式（800px幅 × A4比率）
- 問題ページと解説ページが別々の `.question-section`
- インタラクティブな機能なし
- グレー背景で紙のプレビュー効果

**構造:**
```
問1 問題ページ（.question-section）
├── 問題文（問1．～（配点）の形式）
├── 図表
└── 選択肢

問1 解説ページ（.question-section）
└── 解説

問2 問題ページ（.question-section）
├── 問題文
├── 図表
└── 選択肢

問2 解説ページ（.question-section）
└── 解説
```

**問題文の形式:**
```html
<div class="question-number">問1．ここに問題文を記述します。（6点）</div>
```

**解説の形式:**
```html
<div class="explanation">
    <strong>問1(正解:(X))</strong><br>
    解説内容...
</div>
```

#### template_dynamic.html（インタラクティブ版）

**特徴:**
- 選択肢をクリックすると正解/不正解が表示される
- 解説が自動的に展開される
- プログレスバーで問題の進捗を表示
- 問題番号と配点・時間が別々に表示
- 親フレームに解答を送信する機能（recordAnswer）

**構造:**
```
プログレスバー（.progress-bar）

問1 画面（.screen）
├── ヘッダー（問題番号 + 配点・目安時間）
├── 問題文
├── 図表
├── 選択肢（クリック可能なボタン）
├── 解説セクション（解答後に展開）
└── ナビゲーションボタン

問2 画面（.screen）
├── 同様の構造
```

**問題ヘッダーの形式:**
```html
<div class="question-header">
    <span class="question-number">問1</span>
    <span class="question-meta">配点: 6点 / 目安時間: 3分</span>
</div>
```

**問題文の形式:**
```html
<div class="question-text">
    ここに問題文を記述します。
</div>
```

**選択肢の形式（図形選択肢）:**
```html
<div class="options-figure" id="q1-options">
    <button class="option-figure-button" data-option="1" onclick="selectAnswer(1, 1, 正解番号)">
        <svg>...</svg>
    </button>
    <!-- 選択肢2〜5も同様 -->
</div>
```

**解説の形式:**
```html
<div class="explanation-section" id="q1-explanation">
    <div class="explanation-header">解説（正解: (X)）</div>
    <div class="explanation-content">
        <p>解説内容...</p>
    </div>
</div>
```

**recordAnswer の設定（重要）:**
問題名は `YYYY_MM/航大思考N/問Q` 形式で指定する。ファイル作成時に適切な値に変更すること。
```javascript
window.parent.recordAnswer(
    `2026_01/航大思考X/問${questionNum}`,  // ← ファイルに応じて変更
    selectedOption,
    correctOption,
    isCorrect
);
```

## 図表サイズガイドライン（重要）

### コンテンツ幅の制約

テンプレートの実効コンテンツ幅は**約660px**（800px - padding 40px×2 - 余白）。
**すべての図表はこの幅に収まるように設計すること。**

### SVGサイズ規則

| 項目 | 推奨値 | 最大値 |
|------|--------|--------|
| SVG幅 | 500px | 650px |
| SVG高さ | 300px | 500px |

### 必須設定

1. **viewBox属性を必ず設定する**
   - viewBoxを設定することで、CSSのmax-width: 100%が正しく機能する
   - width/height属性とviewBoxの値を一致させる

```html
<!-- 正しい例 -->
<svg width="500" height="300" viewBox="0 0 500 300">

<!-- 悪い例（viewBoxなし - レスポンシブに対応できない） -->
<svg width="500" height="300">
```

2. **余白を確保する**
   - SVG内の描画要素は、端から最低20pxの余白を確保
   - テキストラベルが切れないよう注意

3. **大きな図が必要な場合**
   - 図を分割するか、スケールダウンを検討
   - 必要に応じてoverflow: visibleを使用しない（はみ出しの原因）

### 作成後の確認手順

1. HTMLファイルをブラウザで開く
2. ウィンドウ幅を800px程度に縮小して確認
3. 図が枠内に収まっているか、テキストが切れていないか確認
4. 印刷プレビューでも確認（実際の試験は紙媒体）

## SVG座標ガイドライン

SVG図表作成時は、各SVGの前にコメントで座標系とサイズを文書化する:

```html
<!--
図表仕様:
- SVGサイズ: 500x300px
- viewBox: 0 0 500 300
- 余白: 上下左右20px

座標系定義:
- 原点: (50, 250)
- X軸: 右方向, 1単位=30px
- Y軸: 上方向, 1単位=40px (SVGは下が正なので減算)
-->
```

作成後はPythonで座標を検証し、軸ラベルとグラフ要素の位置が一致することを確認する。

```python
def verify_solution():
```

- [ ] Python検証で解が唯一であることを確認

- [ ] SVG座標が意図した位置と一致
- [ ] **SVGにviewBox属性が設定されている**

- [ ] 正解番号がランダム化されている
- [ ] 170-310秒で解答可能
- [ ] 複数ステップの推論が必要
- [ ] **印刷用HTML（印刷{N}.html）が作成されている**
- [ ] **航大思考HTML（航大思考{N}.html）が作成されている**

- [ ] **問題一覧.md が更新されている**
- [ ] **README.md が更新されている**

## 禁止事項

- 視覚的矛盾（グラフ値とテキスト値の不一致）
- 自明な解
- 航空専門知識を必要とする問題設計
- Python検証の省略
- 正解番号の手動選択（特に1番や5番への固定）
- フォーマルでない装飾文字の使用（✓、✗、◯、×などの記号）
- **Word/docx形式でのファイル作成（python-docxの使用を含む）**
- **HTML以外の形式での問題ファイル出力**

## 出力形式に関する厳格ルール

**問題ファイルは必ずHTML形式で出力すること。Word（.docx）、PDF、その他の形式での出力は一切禁止。**

- 印刷用: `template_static.html` をベースに `印刷用/印刷{N}.html` を作成
- 航大思考用: `template_dynamic.html` をベースに `航大思考問題/航大思考{N}.html` を作成
- `python-docx` や類似ライブラリを使用したスクリプトを作成してはならない
- リポジトリに `.docx` ファイルを追加してはならない
